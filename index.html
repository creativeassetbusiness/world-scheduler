What the World Scheduler does

The World Scheduler is a system for measuring civilization using physical numbers: calories, water, energy, housing, medicine, materials, labor hours, and transport. It uses safety margins and production data to find real shortages and set real priorities.

Needs → Supply → Shortage

Calculate what people actually need, then compare that need to what is available. When supply is lower than the safety-target, the system finds the shortage in exact numbers.

Tax direction becomes action

When a resource is short, direction is clear: build farms, water systems, greenhouses, factories, transport, or automation.

Labor hours reveal true cost

Goods require production. Production requires labor. Robots lower human labor and free people for higher quality of life.

The philosophy behind the scheduler

This is not redistribution based on politics. It is engineering civilization as a system. You calculate need, add a safety margin, measure supply, find the gap, direct resources, and repeat.

  • Do we have enough calories?
  • Do we have enough water?
  • Do we have enough housing?
  • Do we have enough energy?
  • Do we have enough medicine?
  • Do we have enough materials?
  • Do we have enough production capacity?
  • Do we have enough transportation?

Data integration blueprint

The portal needs a truth layer behind the visuals. Every regional model, life plan, and device reading should land in a structured pipeline with confidence, recency, and verification data.

Source classes

Member plans, world graph nodes, regional supply baselines, device telemetry, partner APIs, and LLM interpretation streams all need their own ingestion path.

  • Profiles and scenarios from the member workspace
  • Region and component metrics from the world graph
  • Ground-device readings from the 1,000,000 node fleet
  • External economics, health, climate, and infrastructure feeds

Truth scoring

Every datapoint should carry a probability of truth, not just a value. That score should drive both planning confidence and UI warnings.

  • Confidence from source reliability
  • Recency decay based on timestamp
  • Verification badge from human or device validation
  • Conflict penalties when sources disagree

Core record shape

Every metric should resolve into a common planning record so the portal can compare life plans, global baselines, and physical infrastructure on the same model.

nodeId metric value unit sourceType confidence observedAt verifiedBy

Immediate build micro-steps

The next delivery should connect the existing UI to a defined data contract instead of adding more display-only panels.

  • Attach confidence fields to scenario saves
  • Map world nodes to structured metrics and provenance
  • Define a telemetry payload for ground devices
  • Introduce a visible conflict or stale-data state

Country of Earth baseline

Only public data is allowed into this ranking. Each country is scored on how completely, consistently, and transparently it publishes the evidence required to model its own reality.

  • Legal baseline: public sources only
  • Accuracy baseline: publication coverage, recency, conflict rate, and source diversity
  • Incentive baseline: the closer a country gets to 100% trusted self-description, the closer it gets to being crowned Country of Earth

How family quota works

The scheduler uses family size, regional needs, health, and activity to build a life plan. Premium quota is possible, but the system flags anything above a normal allowance so the extra is clearly visible.

This is where the world scheduler turns raw numbers into a simple life plan: food, housing, energy, medicine, transport, and luxury.
